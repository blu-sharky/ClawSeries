# ClawSeries: 零人AI短剧公司
**——彻底打通 AI 短剧自动化生产的最后一公里**

## 1. 行业洞察：2026年的制片悖论 (The Production Paradox)
* **数万家“AI公司”的诞生**：
  根据行业统计，过去一年间全球涌现了数万家“AI短剧企业”。然而残酷的真相是：**这些公司引以为傲的“人工+AI”工作流，本身就是完全可以被 AI 彻底取代的。** 它们只是把人类变成了 AI 模型的“搬运工”和整合方。
* **成熟的模型，原始的流水线**：
  我们正处于一个视频生成技术极度成熟的时代，大模型可以在数秒内生成电影级的绝美单镜头。然而，要将这些成熟的“点”连成一部连贯的数十集短剧，这数万家公司的做法依然是极其落后的“人工作坊”模式。
* **致命的“人类沟通成本”**：
  尽管工具满天飞，但不同环节之间依然没有被完全自动化。导演需要向提示词师反复解释镜头意图；制片需要用肉眼在一堆素材中筛选废片；剪辑师需要手工将画面、配音和字幕逐帧对齐。
* **人类成为了流水线上最慢的“路由器”**：
  在极其高效的 AI 节点之间，缓慢的人类操作、审片疲劳和无休止的返工确认，成为了吞噬利润、时间和创意的最大瓶颈。

## 3. 我们的使命 (The Mission)
**ClawSeries 不研发底层视频大模型，我们做全新的”自动化制片工厂”。**

我们将传统 AI 短剧公司中耗时耗力的五个核心岗位，完全替换为自主协作的智能体（Agents）。通过消除”人与人、人与机器的沟通损耗”，**ClawSeries 致力于打通 AI 短剧自动化生产的最后一公里**，让”一句话生成百集短剧”从商业概念变成真正的工业级交付。

## 4. 五大自主智能体架构 (The Pentagram of Agents)
*用毫秒级的 Agent 内部通信，取代漫长的人类沟通会议。*

### 4.1 项目总监 (Project Director Agent)
* **职能**：全局状态管理与流程控制。
* **任务**：接收原始一句话指令，自动拆解数十集制片任务。实时监控各环节进度，调度 API 负载，彻底取代人类制片人的统筹与催稿工作。

### 4.2 总导演 (Chief Director Agent)
* **职能**：剧本与戏剧张力控制。
* **任务**：构建宏大且连贯的世界观。自动在每集结尾设置悬念（Hook），确保长线剧情逻辑自洽，无需人类编剧反复修改大纲。

### 4.3 视觉总监 (Visual Consistency Agent)
* **职能**：视觉资产与一致性锚定。
* **任务**：生成并永久锁定角色的脸部特征、服装特征与场景环境资产。确保角色从第1集到最后1集始终如一，消除人类手动对齐素材的痛苦。

### 4.4 提示词架构师 (Prompt Architect Agent)
* **职能**：跨模型语义翻译。
* **任务**：自动将文学分镜转化为底层视频引擎最易理解的 Prompt 矩阵。它自带“避坑指南”，自动引导 AI 避免画面崩坏，取代反复试错的人类提示词工程师。

### 4.5 自动化剪辑师 (Auto-Editor Agent)
* **职能**：音画压制与情感封装。
* **任务**：基于多模态理解，删除掉多余片段，拼接多个分镜，合理添加 BGM 并对齐压制字幕。

## 5. 核心技术哲学 (Engineering Philosophy)
* **零沟通损耗**：五个 Agent 共享一个全局上下文（Context），导演的意图可以 100% 无损传递给剪辑师，彻底消除人类职场中的沟通误解。
* **全自动质检自愈 (Self-Healing)**：内置视觉审核逻辑。一旦发现生成的素材存在逻辑错误（如多出一根手指），QC Agent 会在后台瞬间驳回并触发重练，全程无需人类介入审片。
* **极致并发 (Hyper-Concurrency)**：突破人类精力的物理极限，支持 50 集短剧同时进入并行渲染与剪辑工作流。

## 7. 本地启动（假后端 + 前端）

说明：当前仓库里的 `backend/` 仍然是 demo / 假后端实现，没有完成 mock/real 环境变量切换。因此现在所谓“假后端”，就是直接启动当前 `backend/`。

### 7.1 先清理残留进程

```bash
lsof -ti tcp:8000 | xargs kill -9 2>/dev/null || true
lsof -ti tcp:3000 | xargs kill -9 2>/dev/null || true
```

### 7.2 启动假后端

在项目根目录执行：

```bash
cd backend
uv run uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

启动后可访问：

- API 根路径：`http://127.0.0.1:8000/`
- Swagger 文档：`http://127.0.0.1:8000/docs`

### 7.3 启动前端

新开一个终端，在项目根目录执行：

```bash
cd frontend
uv run python3 -m http.server 3000 --bind 127.0.0.1
```

启动后访问：

- 前端页面：`http://127.0.0.1:3000/`

### 7.4 联调顺序

1. 先启动后端
2. 再启动前端
3. 打开 `http://127.0.0.1:3000/`
4. 前端会请求 `http://127.0.0.1:8000/api/v1`
